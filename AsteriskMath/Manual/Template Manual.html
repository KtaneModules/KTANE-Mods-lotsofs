On the Subject of Decimals

This may be a life or death situation, but no calculators allowed.

  1. Number D in the display is an approximation of the fraction 1/F, rounded to the nearest 7 decimal places, where F can be any integer between 1 and 999.
  2. Take the Asterisk Number, and divide it by 100 000 to get the number A.

See Appendix * for Asterisk Number reference.

  1. Add A to D to get a new number U.
  2. Find the integer S for which 1/S most closely approximates U.
  3. Submit S as the answer using the dials and the submit button. If an incorrect number is submitted, a strike will be incurred on the bomb.
  4. If the correct number is submitted, the module will disarm and mutate the Asterisk Number by M.
  5. To calculate M, take the difference between F and S, modulo 100. If the result is 0, add 1.

On the Subject of Long Multiplication

A calculator would come in handy for this, but that wouldn't be fun. Exploding isn't fun.

  1. The display shows a number. This is the multiplicand.
  2. Take the Asterisk Number, multiply it by 10, and add the first numeric digit of the serial number. This is the multiplier.

See Appendix * for Asterisk Number reference.

  1. Use long multiplication to find the product.
  2. Take all the digits that were carried during previous step (including the final carry, and carries during the addition part), and add each of them to the product.
  3. Submit this as the answer using the keypad and the submit button. Submitting the wrong number will result in a strike.
  4. Submitting the correct number will disarm the module. The Asterisk Number will be mutated by the sum of all the carried digits.

Long Multiplication

Instructions on how to do long multiplication:

  1. Multiply the number in the ones place of the multiplier by the number in the ones place of the multiplicand. If this results in a two digit number, carry over the leftmost digit into the next step.
  2. Multiply the number in the ones place of the multiplier by the number in the tens place of the multiplicand, and add the potential digit carried from the previous step. Carry the leftmost digit again like before (This would be the digit in the hundreds place).
  3. Also do the above steps for the hundreds, thousands, etc. place of the multiplicand, until there are no numbers left in the multiplicand. Don't forget to add a potential final carry.
  4. Repeat the above steps, but using the number in the tens place of the multiplier, then the hundreds place, etc., until there are no more numbers left in the multiplier.
  5. Use long addition to find the sum of all the numbers calculated in the steps above. First take all the digits in each of the terms' ones places and add them all together. Again, take care to carry digits as before.
  6. Do the above, but for the numbers in the tens place, then the hundreds place, etc. Again, don't forget the final carry. This will yield the final answer.

On the Subject of Adding Then Subtracting

Why would you add and then subtract again? Wouldn't that result in the same number you started with?

  1. Using the right arrow will cycle through the 2-digit numbers in the display. There is no way to go back and view previous numbers.
  2. In total, the module will present five 2-digit numbers. Add all five numbers together.
  3. Add the Asterisk Number as well to get the minuend.

See Appendix * for Asterisk Number reference.

  1. The sixth number shown will be a 3-digit number. This is the subtrahend.
  2. Calculate the difference and use the combination dial to input it.
  3. To submit the answer, press the + button if the difference is positive, or the - button if it is negative. Submitting the incorrect answer will result in a strike.
  4. Submitting the correct answer will disarm the module. This will mutate the Asterisk Number by the positive difference of the second and third number shown in the display.

On the Subject of Spinning Discs

This is making me dizzy, and you're telling me I have to do maths problems while looking at it?

  • The module presents a spinning disc with colored numbers, partially covered up by another spinning disc.
  • Each colored number is accompanied by three buttons. The + button increases it by 1, the - button decreases it by 1, and the R button resets it to its original value. Buttons can be held.
  • The numbers will loop around to 0 if they go above 99, and vice versa.
  • Change each number to its corresponding value, based on its color, initial value, and the Asterisk Number. Use the table to find the correct value.
  • Use the submit button to submit the numbers. Submitting any incorrect number will cause a strike and reset the incorrect numbers.
  • Submitting the right numbers will disarm the module and mutate the Asterisk Number by the number the discs settle on.

Keep Talking and Nobody Explodes Mod Appendix *

Appendix *: The Asterisk Number

  • The Asterisk Number is a number that is the same across all modules on the bomb that use it.
  • The initial value of the Asterisk Number can be found by multiplying the third character of the serial number by 5 and adding the last character of the serial number. If both the third and last character of the serial are 0, the initial value is 30.
  • Modules that use the Asterisk Number can be recognized by a large square panel with a red asterisk scribbled onto it.
  • When a module that uses the Asterisk Number is solved, it will mutate the number. That is, the number will change. The new Asterisk Number will then need to be used for any remaining modules on the bomb that use the number.
  • The Asterisk Number can be reset to its initial value by holding down the square panel on any module. Doing so will incur a strike on the bomb.

Mutation

  • When a module using the Asterisk Number is solved, it will mutate it.
  • The amount with which the number is mutated, depends on the module and can be found on each module's page. The mutation will always occur with a number between 1 and 99.
  • Mutations happen as follows:
    • Initially, the mutation value is added to the Asterisk Number. Eg.: 20 ± 10 = 30.
    • The Asterisk Number cannot be higher than 99. In the event that addition would cause the Asterisk Number to go above 99, the remainder is subtracted instead. Eg.: 95 ± 10 = 93.
    • From then on, any subsequent mutations will be done by subtracting, until the Asterisk Number hits its lower bound of 1, after which it turns back to addition. Eg.: 8 ± 10 = 4.
    • In short, the Asterisk Number will go back and forth between 1 and 99.
